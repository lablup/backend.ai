## SDK v2 Group Client API

### File: src/ai/backend/client/v2/domains/group.py

**Relevant methods for our tests:**

1. **delete(group_id: UUID) -> DeleteGroupResponse**
   - Soft delete operation
   - Returns: `DeleteGroupResponse(deleted: bool)`

2. **search(request: SearchGroupsRequest) -> SearchGroupsResponse**
   - POST /groups/search
   - Request: filter (GroupFilter), order, limit, offset
   - Response: list[GroupDTO] + PaginationInfo

3. **No direct purge method in SDK** - purge is a processor action

## Service Layer Actions

### File: src/ai/backend/manager/services/group/processors.py

**GroupProcessors includes:**
- `delete_group`: Soft delete (DeleteGroupAction)
- `purge_group`: Hard delete (PurgeGroupAction) - validates no vfolder mounts, active kernels, or endpoints
- `search_projects`: Search with filters (SearchProjectsAction)
- `usage_per_period`: Get usage stats (UsagePerPeriodAction) - has start_date/end_date

### Purge Validation (from testspec.py)
The purge operation blocks when:
1. Active endpoints exist
2. Active sessions exist
3. (Implied) Vfolder mounts exist
