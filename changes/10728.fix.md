Fix deployments created via `ActivateRevision` getting stuck in `DEGRADED`. Such deployments bypass `check_pending` (where the appproxy endpoint is normally registered), so the DEPLOYING provisioning handler now registers them itself before route provisioning. Registration and URL persistence are performed atomically per deployment; on persistence failure the endpoint is best-effort unregistered from appproxy so the next coordinator tick does not observe an orphaned endpoint and double-register it.
