Add `max_concurrent_logins` to user_resource_policy to cap the number of simultaneous authenticated login sessions per user. Null means unlimited. This is distinct from `keypair_resource_policies.max_concurrent_sessions` which limits concurrent compute sessions. When the limit is reached, a login without `force=true` is rejected with HTTP 429; a login with `force=true` evicts the oldest sessions to make room for the new one.
