Add an opt-in `[memray]` config section (with an optional `native-traces` mode) for capturing memory allocations to diagnose memory growth in the agent, manager, and storage proxy, backed by a shared `MemrayConfig`. The app-proxy coordinator/worker also converge on the same shared config via a new `[profiling.memray]` table, keeping the previous `enable_memray` / `memray_output_destination` keys working as deprecated aliases. Each forked service worker now closes its own capture on shutdown so the worker-side capture file is no longer truncated.
