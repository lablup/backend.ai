Allow a vfolder referenced by UUID to be mounted at several distinct subpaths within a single session. `prepare_vfolder_mounts` previously re-bucketed UUID-referenced mount requests into per-UUID maps, silently keeping only the last subpath/destination for each vfolder; each request is now preserved so one vfolder can be mounted at multiple subpaths (overlapping sources are still rejected by `is_mount_duplicate`). A UUID-referenced mount request that resolves to no accessible vfolder now raises `VFolderNotFound` instead of being silently dropped.
