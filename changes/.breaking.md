### Breaking Changes
* Change session/service `tag` field from string to structured key-value dictionary throughout the stack. This affects:
  - Database schema: `sessions.tag` and `endpoints.tag` columns increased from varchar(64) to varchar(2048) to store JSON-serialized dictionary
  - Application layer: Tag is now `dict[str, str] | None` instead of `str | None`
  - API: Tag parameter now accepts `dict[str, str]` (e.g., `{"project": "ai", "env": "prod"}`) instead of plain strings
  - CLI: `--tag` option now expects comma-separated key=value pairs (e.g., `--tag project=ai,env=prod`)
  - Client SDK: All session/service creation functions now require dict type for tag parameter

The application automatically serializes/deserializes tags between dict and JSON string format. Existing string tags are automatically wrapped as `{"value": "<old_tag>"}` when loaded for backward compatibility. ([#1851](https://github.com/lablup/backend.ai/pull/1851))
