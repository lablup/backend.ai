Derive the `--changed-since` base ref from the PR merge commit's first parent (`HEAD^1`) in `ci.yml`, so pants can always locate a merge base without extra fetches — including for stale (un-rebased) PRs and PRs that have merged `main` back into the topic branch. As a result the per-job fetch-depth calculation collapses to a constant `fetch-depth: 2` (the merge commit plus its two parents), dropping the `pull_request.commits`-based math and the full-history fetch for fork PRs.
